## 개발 목적

CodexCode CLI worker를 관리하는 외부 오케스트레이션 프레임워크(Coreline Orchestrator) v1을 구현한다.
HTTP API로 Job을 생성하고, process-based worker를 spawn하며, worktree 격리·로그 수집·결과 집계·SSE 이벤트를 제공하는 단일 호스트 시스템을 완성한다.

## 개발 범위

- 프로젝트 스캐폴딩 (package.json, tsconfig.json, 디렉토리 구조)
- Core domain 타입, 상태 머신, ID 생성, 에러, 이벤트
- 파일 기반 StateStore (`.orchestrator/` 하위 JSON/NDJSON)
- Config 로딩, repo allowlist, git worktree 생명주기
- CodexCode CLI process spawn RuntimeAdapter, 로그 수집
- Worker 생명주기 관리, worker-authored result JSON 수집 및 job 결과 집계
- FIFO + priority Scheduler, capacity/conflict 정책, job별 multi-worker fan-out
- Hono HTTP API (`/api/v1/jobs`, `/workers`, `/artifacts`, `/health`, `/metrics`), SSE 이벤트
- Reconciliation, cleanup, cancellation, retry, graceful shutdown

## 제외 범위

- 분산 multi-host 실행
- Session-aware / direct-connect adapter
- WebSocket 전송
- Object storage backend
- Multi-tenant auth / public internet hardening
- CodexCode CLI 내부 수정
- UI 또는 프론트엔드
