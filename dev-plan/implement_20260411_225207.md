# implement_20260411_225207.md

작성 일시: `2026-04-11 22:52:07 KST`

이 문서는 distributed prototype 이후 Coreline Orchestrator의 다음 개발 범위를 고정하고, production-grade distributed control plane 전환 작업이 목적 밖으로 확장되지 않도록 하기 위한 작업 문서다.

## 개발 목적

현재 sqlite coordinator + sqlite queue + polling event stream + manifest transport 기반 prototype을 production-grade distributed control plane으로 끌어올리기 위한 다음 개발 순서를 고정한다.

- prototype seam을 external/shared infrastructure로 치환 가능한 방식으로 구체화한다.
- queue / event / artifact transport를 실제 multi-host 운영 가능한 backend로 전환한다.
- remote executor를 local simulation이 아닌 network worker-plane으로 연결한다.
- cutover / rollback / failover 절차를 production readiness 수준으로 고정한다.

## 개발 범위

- external coordinator service 또는 equivalent shared coordination backend contract
- broker-backed durable dispatch queue / event stream
- network object-store artifact/log/result/transcript transport
- remote executor claim / heartbeat / result publish network path
- distributed cutover / rollback / failover / degraded-mode readiness

## 제외 범위

- browser admin console
- multi-region active-active replication
- SaaS tenant/billing/orchestration UI
- codexcode 자체 프로토콜 변경

## 잔여 리스크 / 후속 과제

- external coordinator backend 선택(예: Redis/service/DB)에 따라 failover latency와 fencing semantics가 달라질 수 있다.
- broker/object-store cutover 이후에도 local/sqlite/manifest fallback을 유지해야 한다.
- network worker-plane 도입 후 auth/audit boundary를 executor 간 transport까지 확장해야 한다.
